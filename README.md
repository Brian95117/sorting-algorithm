## 📌 專案介紹

本專案為一個 **排序演算法視覺化與效能分析系統**，透過互動式網頁展示排序過程，並結合即時統計與圖表分析，讓使用者能直觀理解不同排序方法的運作機制與效能差異。

與傳統靜態教學相比，本系統強調「**互動 + 視覺化 + 數據分析**」，提升學習演算法的理解深度。

---

## ✨ 專案特色

| 排序法 | 英文 | 時間複雜度 |
|--------|------|------------|

## 📊 實驗分析

### 🔍 實驗設計

- 測試不同資料規模（n = 10 ~ 200）
- 使用隨機資料進行排序
- 每個排序法重複執行多次取平均值
- 比較執行時間與趨勢變化

---

### 📈 實驗結果觀察

1. **O(n²) 演算法（Bubble / Selection / Insertion）**
   - 當 n 增加時，執行時間呈現快速上升
   - 特別在 n ≥ 100 時差異明顯

2. **O(n log n) 演算法（Merge / Heap）**
   - 成長曲線較平緩
   - 在大資料情況下明顯優於 O(n²)

3. **資料分布影響**
   - 插入排序在「接近排序」時表現顯著提升
   - 氣泡排序最佳情況也可接近 O(n)

---

## 📌 結論

- 小規模資料（n < 30）：
  - 插入排序具有不錯效能
  - 實作簡單，適合教學用途

- 中大型資料：
  - Merge Sort 效能穩定且可預測
  - Heap Sort 在空間效率上更具優勢

- 綜合比較：
  - 時間複雜度為選擇演算法的關鍵因素
  - 視覺化能有效幫助理解演算法行為

## 💡 心得

在本專案開發過程中，最大的收穫不只是實作排序演算法，而是透過「視覺化」真正理解演算法的運作方式。

以往在學習排序時，僅透過程式碼或理論公式（如 O(n²)、O(n log n)）較難直觀感受到差異，但透過動畫呈現與數據統計，可以清楚看到：

- 比較次數如何累積  
- 元素如何移動  
- 效率差距如何隨資料量放大  

此外，在實作過程中也發現：

- Merge Sort 雖然效率高，但需要額外空間  
- Heap Sort 雖然複雜，但在空間上更具優勢  
- 插入排序在特定情況下其實非常有效  

這讓我對「演算法選擇」有更實務性的理解，而不是單純記憶公式。
