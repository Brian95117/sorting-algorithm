## 氣泡排序法 (Bubble Sort)

### 原理
氣泡排序透過重複掃描陣列，比較相鄰元素，若順序錯誤則交換，使較大的元素逐步「冒泡」到陣列尾端。

### 複雜度
- 最佳：O(n)
- 平均：O(n²)
- 最差：O(n²)
- 空間：O(1)

### 特點
- 實作簡單
- 效率較低
- 穩定排序

---

## 選擇排序法 (Selection Sort)

### 原理
每一輪從未排序區間中找出最小值，與當前位置交換，使已排序區間逐步擴大。

### 複雜度
- 最佳 / 平均 / 最差：O(n²)
- 空間：O(1)

### 特點
- 交換次數較少
- 不穩定排序
- 不受資料初始順序影響

---

## 插入排序法 (Insertion Sort)

### 原理
將資料分為已排序與未排序區間，逐一取出元素並插入到已排序區間的正確位置。

### 複雜度
- 最佳：O(n)
- 平均：O(n²)
- 最差：O(n²)
- 空間：O(1)

### 特點
- 對接近排序資料效率高
- 穩定排序
- 適合小型資料

---

## 合併排序法 (Merge Sort)

### 原理
採用分治法，將陣列分割為子陣列，分別排序後再合併成完整排序結果。

### 複雜度
- 最佳 / 平均 / 最差：O(n log n)
- 空間：O(n)

### 特點
- 效率穩定
- 適合大型資料
- 需額外空間
- 穩定排序

---

## 堆積排序法 (Heap Sort)

### 原理
利用最大堆結構，每次將最大值移到尾端，並重新調整堆結構。

### 複雜度
- 最佳 / 平均 / 最差：O(n log n)
- 空間：O(1)

### 特點
- 不需額外空間
- 效率穩定
- 不穩定排序

---

## 綜合比較

| 排序法 | 時間複雜度 | 空間 | 穩定性 | 特性 |
|--------|------------|------|--------|------|
| Bubble | O(n²) | O(1) | 穩定 | 簡單但效率低 |
| Selection | O(n²) | O(1) | 不穩定 | 交換少 |
| Insertion | O(n²) | O(1) | 穩定 | 小資料效率佳 |
| Merge | O(n log n) | O(n) | 穩定 | 大資料穩定 |
| Heap | O(n log n) | O(1) | 不穩定 | 空間效率佳 |

## 學習心得

透過本次排序演算法的實作與視覺化模擬，我對演算法的理解從單純的理論，進一步提升到實際運作層面。

在未進行模擬前，對於 O(n²) 與 O(n log n) 的差異僅停留在公式上，但透過動畫與實驗測試，可以清楚觀察到當資料量增加時，排序時間的成長速度差異非常明顯。

此外，在不同資料分布情況下，也觀察到：

- 插入排序在接近排序資料時效率顯著提升  
- 氣泡排序在最佳情況下也能達到 O(n)  
- Merge Sort 與 Heap Sort 在大資料時明顯優於其他方法  

這些結果讓我了解到，實際應用中選擇排序法時，不能只看理論複雜度，還需考慮資料特性與使用場景。

---

## 結論

本次實驗比較五種排序演算法後，可得以下結論：

1. O(n²) 演算法在資料量大時效率不佳，不適合實務應用  
2. O(n log n) 演算法在大數據下表現穩定，是較佳選擇  
3. 插入排序在小型或接近排序資料中仍具有實用價值  
4. Merge Sort 與 Heap Sort 各有優勢：
   - Merge Sort：穩定但需額外空間  
   - Heap Sort：節省空間但較複雜  

整體而言，排序演算法的選擇需依據資料規模與需求進行權衡，而非單一最佳解。
